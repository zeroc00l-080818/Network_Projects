## 1. Project Overview

**Client:** Radeon Company Ltd. (Banking & Insurance)
**Location:** Nairobi, Kenya (African branch)
**Building:** 4-story office building
**Simulation Tool:** Cisco Packet Tracer

Radeon Company Ltd. requires a fully functional enterprise network for its first African branch. The network must support multiple departments across four floors, provide wired and wireless connectivity, enforce security policies, and be resilient to single points of failure through a hierarchical redundant design.

---

## 2. Network Topology Design

The network follows the **three-layer hierarchical model**:

| Layer | Devices Used | Purpose |
|---|---|---|
| **Core** | Cisco 2911 Routers (1 per floor) | Inter-floor routing via Serial DCE links |
| **Distribution** | Cisco 3650 L3 Switches (2 per floor) | InterVLAN routing, redundancy, DHCP relay |
| **Access** | Cisco 2960 L2 Switches (1 per department) | End-device connectivity, port security |

### Per-Floor Topology

```
  [2911 Router] ←— Serial DCE —→ [Other Floor Routers]
       |              |
  [L3-SW-A]      [L3-SW-B]          ← Redundant pair (Distribution)
    /  |  \       /  |  \
  L2  L2  L2   L2  L2  L2           ← One per department (Access)
  |    |    |    |    |    |
 PCs  PCs  PCs  PCs  PCs  PCs       ← End devices + Printers + APs
```

> Both L3 switches connect to ALL L2 switches on their floor to ensure redundancy. If one L3 switch fails, the other maintains connectivity.

### Inter-Router Serial Connections (Core Layer — Ring Topology)

```
[R1 - Floor 1] —Se—→ [R2 - Floor 2]
      ↑                      ↓
[R4 - Floor 4] ←—Se— [R3 - Floor 3]
```

The four routers form a **ring** (R1↔R2, R2↔R3, R3↔R4, R4↔R1) via Serial DCE links, providing a redundant path in the core layer.

### Router ↔ L3 Distribution Switch Connections

```
R1 ──→ MS1,  R4 ──→ MS1      (MS1 serves Floors 1 & 4)
R1 ──→ MS2,  R4 ──→ MS2      (MS2 serves Floors 1 & 4)
R2 ──→ MS3,  R3 ──→ MS3      (MS3 serves Floors 2 & 3)
R2 ──→ MS4,  R3 ──→ MS4      (MS4 serves Floors 2 & 3)
```

Each pair of multilayer switches (MS1/MS2 and MS3/MS4) is connected to two routers for redundancy, ensuring continuity if one router fails.

> OSPF is used across all routers and L3 switches to share routing information dynamically.

## 6. Step-by-Step Configuration Guide

---

### 6.1 Basic Configuration — L2 Switches

> Repeat this for every L2 (2960) switch. Replace `[name]` and `[password]` with actual values.

```
> en
# conf t
# hostname [DeptName-L2-SW]
# banner motd # Authorized Access Only #
# no ip domain-lookup
# enable secret [password]
# service password-encryption
# line console 0
#  password [password]
#  login
#  exit
# line vty 0 15
#  password [password]
#  login
#  exit
# do wr
```

---

### 6.2 Basic Configuration & SSH — L3 Switches and Routers

> SSH must be configured on **all routers and L3 switches** as per project requirements.

```
> en
# conf t
# hostname [DeviceName]
# banner motd # Authorized Access Only #
# no ip domain-lookup
# enable secret [password]
# service password-encryption
# ip domain-name radeon.com
# username admin password [password]
# crypto key generate rsa
  → Enter 1024 when prompted for modulus size
# line console 0
#  password [password]
#  login
#  exit
# line vty 0 15
#  login local
#  transport input ssh
#  exit
# do wr
```

---

### 6.3 VLAN & Port Security — L2 Switches

> Configure the access ports connected to end devices. Check which ports are actually connected before using the range command.

```
> en
# conf t
# vlan [VLAN_ID]
#  name [DepartmentName]
#  exit
# int range fa0/2-24
#  switchport mode access
#  switchport access vlan [VLAN_ID]
#  switchport port-security
#  switchport port-security maximum 2
#  switchport port-security mac-address sticky
#  switchport port-security violation shutdown
#  exit
# do wr
```

> `maximum 2` allows one PC and one printer per port. Adjust as needed per port.

---

### 6.4 Trunk Port Configuration — L2 Switches

> The uplink port(s) from each L2 switch going to L3 switches must be in trunk mode to carry all VLANs.

```
> en
# conf t
# int range fa0/1-2
#  switchport mode trunk
#  exit
# do wr
```

> Check physical connections and update the interface range accordingly.
> On the L3 switch side, the downlink ports to L2 switches must also be set to trunk mode.

---

### 6.5 IP Addressing — Routers & L3 Switches

**On each Router — Serial interfaces (DCE side requires clock rate):**

```
> en
# conf t
# int se0/0/0
#  ip address [IP] [SubnetMask]
#  clock rate 64000          ← Only on DCE end
#  no shutdown
#  exit
# do wr
```

**On L3 Switches — Convert port to Layer 3 before assigning IP:**

```
> en
# conf t
# int gig0/0
#  no switchport
#  ip address [IP] [SubnetMask]
#  no shutdown
#  exit
# ip routing
# do wr
```

> `no switchport` converts a Layer 2 port to a Layer 3 routed port. This must be done before assigning an IP address on the L3 switch physical interfaces.

---

### 6.6 OSPF Configuration

**On Routers:**

```
> en
# conf t
# router ospf 1
#  network [NetworkAddress] [WildcardMask] area 0
#  ← Repeat for every directly connected network
#  passive-interface [interface]   ← Apply on all user-facing interfaces
#  exit
# do wr
```

**Example for R1 (Floor 1 Router):**

```
# router ospf 1
#  network 10.10.10.0 0.0.0.3 area 0       ← R1–R2 serial link
#  network 10.10.10.12 0.0.0.3 area 0      ← R4–R1 serial link
#  network 10.10.10.16 0.0.0.3 area 0      ← R1–MS1 link
#  network 10.10.10.28 0.0.0.3 area 0      ← R1–MS2 link
#  exit
```

**On L3 Switches (enable IP routing first):**

```
> en
# conf t
# ip routing
# router ospf 1
#  network [NetworkAddress] [WildcardMask] area 0
#  ← List all VLAN SVI networks and router-facing link networks
#  exit
# do wr
```

### 6.8 InterVLAN Routing & DHCP Helper — L3 Switches

> Both L3 switches on each floor must have the same VLANs created and the same SVI IP addresses and DHCP helper addresses configured.

**Step 1 — Create VLANs on L3 Switch:**

```
> en
# conf t
# vlan 10
# vlan 20
# vlan 30
  ← Continue for all VLANs relevant to this floor
# exit
```

**Step 2 — Assign SVI IPs and DHCP Helper:**

```
# int vlan 10
#  ip address 192.168.1.1 255.255.255.192
#  ip helper-address 192.168.3.193    ← IP of the DHCP server
#  no shutdown
#  exit
# int vlan 20
#  ip address 192.168.1.65 255.255.255.192
#  ip helper-address 192.168.3.193
#  no shutdown
#  exit
# int vlan 30
#  ip address 192.168.1.129 255.255.255.192
#  ip helper-address 192.168.3.193
#  no shutdown
#  exit
  ← Continue for all VLANs (40 through 120) using addresses from Section 5
# do wr
```

> **Important:** The `ip helper-address` command forwards DHCP broadcast requests from devices to the centralized DHCP server in the server room. Without this, devices will not receive IPs automatically.

## 7. Known Issues & Recommended Fixes

The following issues were identified in the original configuration steps. Each should be corrected before or during implementation.

---

### 7.1 Typos in CLI Commands

| # | Location | Incorrect | Correct |
|---|---|---|---|
| 1 | Step 5 | `#conf that` | `#conf t` |
| 2 | Step 3 | `#siwtchport port-security violation shutdown` | `#switchport port-security violation shutdown` |
| 3 | Step 5 (wildcard note) | `255.255.255.255.192` | `255.255.255.192` |

---

### 7.2 Missing Commands

| # | Location | Issue | Fix |
|---|---|---|---|
| 4 | Step 4 & 5 | `no shutdown` missing after IP assignment | Add `no shutdown` on every interface after assigning IP |
| 5 | Step 4 | `clock rate` missing on DCE serial interfaces | Add `clock rate 64000` on the DCE end of all serial links |
| 6 | Step 3 | Port-security maximum has no value | Change to `switchport port-security maximum 2` (or 1 as needed) |
| 7 | Step 5 | OSPF process ID missing | Change `router ospf` → `router ospf 1` |
| 8 | Step 6.8 | `no shutdown` missing on VLAN SVIs | Add `no shutdown` after assigning each SVI IP |
| 9 | Step 7 | VLANs not created on L3 switch before SVI assignment | Create VLANs with `vlan [ID]` before entering `int vlan [ID]` |

---

### 7.3 Security Issues

| # | Issue | Fix |
|---|---|---|
| 10 | `enable password` is used — stores password in plaintext | Replace with `enable secret [password]` which uses MD5 hashing |
| 11 | SSH not configured on 2911 Routers (Step 2.2 only covers L3 switches) | Apply the full SSH configuration block (Step 6.2) to all 4 routers as required |

---

### 7.4 Design & Logic Issues

| # | Issue | Recommendation |
|---|---|---|
| 12 | Step 6 says "go to IT department DHCP server" | DHCP server is in the **Server Room** on Floor 4, not the IT department |
| 13 | VLAN list in Step 7 only goes up to VLAN 60 | The network has **12 departments** — VLANs must go up to 120 (10 through 120 in steps of 10) |
| 14 | No redundancy protocol for L3 switch pair | Implement **HSRP** on VLAN SVIs so one switch is Active and the other is Standby. Without HSRP, both switches will have the same default gateway IP causing conflicts and OSPF router-ID collisions |
| 15 | No OSPF passive interfaces configured | Add `passive-interface` for all interfaces facing end devices to prevent unnecessary OSPF hellos on user ports |
| 16 | No trunk VLAN filtering on L3 switch uplinks | Use `switchport trunk allowed vlan [list]` to restrict VLANs on each trunk and reduce broadcast traffic |
| 17 | Guest Area has 40 PCs + wireless | With 40 PCs, 2 printers, and wireless users, the total could exceed 62. Monitor actual user count; upgrade to /25 (126 hosts) if needed |
| 18 | Server Room VLAN SVI acts as default gateway | Manually configure static IPs on all 3 servers pointing to the Server Room SVI as their default gateway |

## 8. Testing & Verification Checklist

Use the following commands to verify the network is working correctly:

### On Routers / L3 Switches

```
# show ip interface brief          → Check all interfaces are up/up
# show ip route                    → Verify OSPF routes are learned
# show ip ospf neighbor            → Verify OSPF adjacencies formed
# show vlan brief                  → Verify VLANs exist
# show interfaces trunk            → Verify trunk ports and VLANs allowed
# show ip dhcp binding             → View assigned IPs from DHCP
# show port-security interface fa0/x  → Verify port security per port
# show running-config              → Review full configuration
```

### End Device Tests (Packet Tracer PDU / CLI Ping)

| Test | Expected Result |
|---|---|
| PC in VLAN 10 pings PC in VLAN 10 (same floor) | ✅ Success |
| PC in VLAN 10 pings PC in VLAN 40 (different floor) | ✅ Success via OSPF |
| PC receives IP automatically via DHCP | ✅ IP from correct pool |
| PC accesses HTTP Server via browser | ✅ Web page loads |
| PC sends email via Email Server | ✅ Email delivered |
| SSH from Admin PC to Router | ✅ SSH login successful |
| Unauthorized device on access port triggers violation | ✅ Port shuts down |
